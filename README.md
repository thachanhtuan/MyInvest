# MyInvest — Учёт личных инвестиций

Локальное веб-приложение для учёта инвестиционного портфеля. Работает в браузере, данные хранятся на вашем компьютере в SQLite.

## Возможности

- **Импорт из Excel** — загрузка счетов, активов, облигаций, транзакций и стоимости из многолистового `.xlsx`-файла
- **Дашборд** — общая стоимость портфеля, прибыль/убыток, donut-диаграмма аллокации по классам активов
- **Баланс по счетам** — стоимость, прибыль и доля каждого счёта в портфеле
- **Баланс по брокерам** — агрегация по брокерам/банкам
- **Позиции** — таблица всех открытых позиций с количеством, средней ценой, вложениями, текущей стоимостью и доходностью
- **Просмотр данных** — таблицы счетов, активов, транзакций с фильтрами
- **Экспорт в Excel** — выгрузка текущего баланса в `.xlsx` с 4 листами
- **Swagger UI** — документация всех API-эндпоинтов на `/docs`

## Требования

- Python 3.12+
- [uv](https://docs.astral.sh/uv/) — менеджер пакетов

## Установка и запуск

```bash
# Клонировать репозиторий
git clone <url> && cd MyInvest

# Установить зависимости
uv sync

# Применить миграции (создаёт myinvest.db)
uv run alembic upgrade head

# Запустить сервер
uv run uvicorn app.main:app --reload
```

Приложение доступно по адресу **http://localhost:8000**

## Быстрый старт

1. Откройте http://localhost:8000/import
2. Нажмите **«Импортировать»** — данные из `data-xlsx/data.xlsx` загрузятся в базу
3. Перейдите на **Дашборд** (http://localhost:8000) — увидите сводку портфеля

## Импорт своих данных

Подготовьте Excel-файл (`.xlsx`) с листами по формату, описанному в [docs/EXCEL_FORMAT.md](docs/EXCEL_FORMAT.md). Загрузите через страницу импорта или замените `data-xlsx/data.xlsx`.

## API-эндпоинты

| Метод | URL                      | Назначение                                                       |
| ----- | ------------------------ | ---------------------------------------------------------------- |
| GET   | `/`                      | Дашборд                                                          |
| GET   | `/accounts`              | Страница счетов                                                  |
| GET   | `/assets`                | Страница активов                                                 |
| GET   | `/transactions`          | Страница транзакций (фильтры: `account_id`, `ticker`, `tx_type`) |
| GET   | `/import`                | Страница импорта                                                 |
| POST  | `/api/import/default`    | Импорт из `data-xlsx/data.xlsx`                                  |
| POST  | `/api/import/excel`      | Импорт загруженного файла                                        |
| GET   | `/api/export/balance`    | Скачать баланс в Excel                                           |
| GET   | `/api/accounts/`         | JSON: список счетов                                              |
| GET   | `/api/assets/`           | JSON: список активов                                             |
| GET   | `/api/transactions/`     | JSON: транзакции (фильтры)                                       |
| GET   | `/api/analytics/summary` | JSON: полная аналитика портфеля                                  |
| GET   | `/docs`                  | Swagger UI                                                       |

## Решение частых проблем

### `table accounts already exists` при `alembic upgrade head`

Возникает, если таблицы уже были созданы напрямую (например, при первом запуске приложения), но запись в `alembic_version` отсутствует. Исправление — проставить маркер вручную:

```bash
uv run alembic stamp head
uv run alembic upgrade head   # теперь пройдёт чисто
```

### Сброс БД с нуля

```bash
del myinvest.db                    # Windows
# или: rm myinvest.db             # bash
uv run alembic upgrade head        # создаёт свежую БД
```
