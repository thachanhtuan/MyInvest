# Формат Excel-файла

Основной источник данных — многолистовой Excel-файл (`.xlsx`).  
Каждый лист содержит отдельный тип данных. Имена листов **фиксированы**.

## Обзор листов

| Лист              | Обязательный | Модель             | Описание                      |
| ----------------- | :----------: | ------------------ | ----------------------------- |
| Счета             |      да      | `Account`          | Брокерские / банковские счета |
| Активы            |      да      | `Asset`            | Справочник активов            |
| Облигации         |     нет      | `BondInfo`         | Параметры облигаций           |
| Транзакции        |      да      | `Transaction`      | Все операции                  |
| Котировки         |     нет      | `Quote`            | Цены закрытия                 |
| Купоны            |     нет      | `CouponPayment`    | Расписание купонов            |
| Стоимость актива  |     нет      | `AssetValuation`   | Стоимость по данным брокера   |
| Целевая аллокация |     нет      | `TargetAllocation` | Целевые пропорции             |

## Лист «Транзакции»

| Столбец            | Тип    | Обязательный | Описание                   | Пример       |
| ------------------ | ------ | :----------: | -------------------------- | ------------ |
| `date`             | дата   |      да      | Дата операции              | `2024-03-15` |
| `account_id`       | строка |      да      | Ссылка на `Счета.id`       | `tink1`      |
| `ticker`           | строка |      да      | Ссылка на `Активы.tiker`   | `SBERP`      |
| `tx_type`          | строка |      да      | Тип (см. TransactionType)  | `buy`        |
| `quantity`         | число  |     нет      | Количество                 | `10`         |
| `price`            | число  |     нет      | Цена за штуку              | `265.50`     |
| `amount`           | число  |     нет      | Стоимость актива           | `2655.00`    |
| `nominal_amount`   | число  |     нет      | Сумма в валюте номинала    | `28.50`      |
| `nominal_currency` | строка |     нет      | Код валюты номинала        | `USD`        |
| `nkd`              | число  |     нет      | Накопленный купонный доход | `2658.99`    |
| `commission`       | число  |     нет      | Комиссия по сделке         | `3.99`       |
| `total_amount`     | число  |     нет      | Итоговая сумма с комиссией | `2658.99`    |
| `currency`         | строка |     нет      | Валюта                     | `RUB`        |
| `broker`           | строка |     нет      | Название брокера           | `Т-Инвест`   |
| `notes`            | строка |     нет      | Примечания                 |              |

> **Правило `amount`**: Если `amount` заполнен, именно он используется как стоимость актива (без комиссии). Если пуст — вычисляется как `quantity × price`.
>
> **Правило `total_amount`**: Итоговая сумма с учётом комиссии. Если заполнен — используется напрямую для расчёта затрат (`total_cost += total_amount` при покупке). Если пуст — вычисляется как `amount + commission+nkd` (покупка) или `amount − commission` (продажа).
>
> **Правило `commission`**: Комиссия учитывается при расчёте `total_amount`, если `total_amount` не задан явно.
>
> **Правило `initial_balance`**: Первая транзакция для актива, задающая начальное количество и стоимость. Если отсутствует — начальный баланс = 0.

## Лист «Стоимость актива»

| Столбец            | Тип    | Обязательный | Описание                                                        | Пример       |
| ------------------ | ------ | :----------: | --------------------------------------------------------------- | ------------ |
| `date`             | дата   |      да      | Дата фиксации стоимости                                         | `2024-03-31` |
| `ticker`           | строка |      да      | Тикер актива (ссылка на «Активы»)                               | `SBER`       |
| `value`            | число  |     нет      | Общая стоимость по данным брокера (в указанной валюте)          | `28500.00`   |
| `currency`         | строка |     нет      | Код валюты стоимости (`RUB`/`USD`/`EUR`/`CNY`)                  | `USD`        |
| `nominal_value`    | число  |     нет      | Стоимость в валюте номинала                                     | `310.25`     |
| `nominal_currency` | строка |     нет      | Код валюты номинала (`RUB`/`USD`/`EUR`/`CNY`)                   | `USD`        |
| `value_pct`        | число  |     нет      | Стоимость в процентах (например, `% от номинала` для облигаций) | `101.25`     |

> **Правило `value`**: если `value > 0`, стоимость берётся напрямую.
>
> **Правило `value_pct` для облигаций**: если `value` не задан и `value_pct > 0`, стоимость рассчитывается как `количество на дату × номинал × value_pct / 100`.
>
> **Правило `value_pct` для остальных активов**: если `value` не задан и `value_pct > 0`, стоимость рассчитывается как `% от вложений на дату`.
>
> **Назначение**: позволяет отслеживать динамику стоимости и прибыли по каждому активу в разрезе периодов времени. Стоимость вносится пользователем из данных брокера на определённые даты (например, конец месяца). Аналитика сравнивает с нарастающим итогом инвестиций из транзакций.
